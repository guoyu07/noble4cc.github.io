Python2和Python3的区别总结

Veröffentlicht am 2016-01-16   |   in Python   |  

许多初学Python的同学总是在问一个问题，是该学习Python2还是该学习Python3。许多对这个问题的回答经常是：仅仅学习你就喜欢的版本之后再找出之间的不同就可以了。
但是如果你开发一个新的项目该如何选择版本呢？只要Python2和Python都支持你所需要的类库，没有什么正确和错误的。然而，弄清两者之间的不同，让你在写代码当时避免一些深坑是非常值得的。

这是一个非常危险的变化，因为在Python2中整数之间的除法是整除，而在Python3中会出现小数字。如果在Pytho3中使用整除必须使用两个除号。

#####xrange
在Python2中，xrang()会产生一个可迭代的对象，类似于generator，为什么使用可迭代对象呢，因为当使用for循环进行迭代时，rang方法返回的是一个list，如果数据量过大，list对象都会载入内存，会浪费大量的资源。使用可迭代对象不会产生这个现象。
在Python3中rang默认就是返回可迭代对象，所以xrang在Python3中被取消了。

PYTHON 2

PYTHON 3

Python 3.4.1
timing range()
1000 loops, best of 3: 520 µs per loop

不少人指出Python3的rang方法比Python2的xrang方法要慢，但是应该指出的是：相同的Python实现，2是要比3要快

####for循环变量和全局命名空间的泄露

好消息就是在Python3的for循环变量不在泄露在全局名称空间内了。
可以理解为List在一不支持[… for var in item1, item2, …].语法了，而是被[… for var in (item1, item2, …)]代替了。就是在语法层面上实现了一个list的生成器，使得变量不在泄露到全局空间中。
